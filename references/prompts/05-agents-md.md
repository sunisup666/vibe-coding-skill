# 提示词⑤:生成 AGENTS.md

产出一份 `AGENTS.md`,它是 AI agent 在本项目里干活时必须遵守的操作手册与护栏,放在项目根目录,agent 每次工作都会读它。基于全部已生成文档产出。

## 硬性约束

- 面向 agent 写,指令明确、可执行,不写给人看的背景介绍。
- 所有约束都要能被 agent 直接照做,不含糊。

## 必须包含的章节

1. **项目一句话说明**
2. **绝对禁止事项** —— 从各文档的"不做事项""敏感字段黑名单"提炼成明确清单
3. **唯一事实源约定** —— 声明 `docs/03-data-model.md` 为唯一字段权威,任何数据结构必须对齐它,禁止另立字段
4. **代码风格与分层约定** —— 目录结构、分层依赖方向、命名规范、错误处理方式(对齐技术架构文档)
5. **测试要求** —— 先写测试再写实现;每个功能必须有单测;测试数据用脱敏样例,禁止放真实敏感信息
6. **每个切片完成后必须运行的验证命令** —— 给出具体命令(如 `pytest xxx/tests` 全绿 + 主程序可启动)
7. **何时必须停下来问人**(而不是自己猜)—— 文档未定义的字段/接口结构、需要真实外部数据、需求歧义、要引入新依赖时
8. **范围纪律** —— 一次只做当前指派的切片,禁止顺手改动其他切片范围
9. **提交约定**(如需要)—— commit 粒度、信息格式

## 输出后自检

末尾附自检:
- 禁止事项是否都从文档如实提炼、无遗漏
- 验证命令是否具体可执行
- "何时停下问人"是否覆盖了 agent 容易擅自猜测的场景

## 呈现给用户时

主动提示:禁止事项有没有漏、验证命令对不对、"何时问人"够不够。
