# 提示词⑥:逐切片开工(每次只做一个切片)

实现代码阶段。以全部文档(`docs/01`~`docs/04`)+ `AGENTS.md` 为上下文,先阅读并严格遵守 `AGENTS.md`。**一次只实现一个切片。**

## 工作要求

1. 严格对齐 `docs/03-data-model.md` 的字段定义,禁止自创或改写字段。
2. 先写测试,再写实现(TDD)。测试要覆盖该切片"完成判定"里的所有断言。
3. 只做当前切片在 `docs/04-slices.md` 里划定的范围,禁止改动其他切片的文件/逻辑。
4. 完成后运行本切片规定的验证命令,必须全绿;把运行结果贴出来给用户看。
5. 遇到以下情况必须停下来问用户,不许自己猜:
   - 文档里没定义的字段、结构或接口
   - 需要真实外部数据/样例但用户还没提供
   - 需求有歧义或多种理解
   - 需要引入文档未提及的新依赖
6. 完成后给一份简短交付说明:改了哪些文件、验证命令与结果、有没有遗留问题。

## 依赖外部真实数据的切片

如果该切片需要外部真实数据(爬取响应、第三方接口、外部文件格式),而用户还没提供"真实输入 + 期望输出"样例,**停下来先要样例,不要猜着写 parser。** 这类东西 agent 猜不出来。

## 每个切片完成后

让用户确认绿灯通过,再进入下一个切片。不要连续做多个切片不给用户确认。
